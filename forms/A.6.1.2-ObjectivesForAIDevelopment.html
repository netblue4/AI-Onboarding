A.6.1.2 Objectives for AI Development

This document outlines the objectives for responsible AI development, including how these objectives will be measured, evidenced, and the operational duties involved.

Objectives for AI Development (A.6.1.2)

Principle Objective How It Will Be Measured/Evidenced Testing Operational Duties Responsible Accountable Informed
Human-Centric AI Design the AI system to augment human roles where feasible and identify functions requiring workforce adaptation.
  • AI Impact Assessment identifying potentially affected roles
  • Design documents showing human-in-the-loop integration points
  • Early-stage workforce transition and retraining plans
 Plan for Transition: Develop an initial workforce transition plan based on the system's design and anticipated impact.
Fairness Build and validate a model that demonstrates equitable performance across key demographic subgroups.
  • Pre-deployment fairness test results across protected attributes
  • Bias mitigation strategy documents and validation outcomes
  • Data sourcing and representativeness analysis
 Build for Fairness: Incorporate fairness metrics into the model development and validation pipeline. Select and apply appropriate bias mitigation techniques before finalizing the model.
Transparency Develop a system whose decision-making processes are understandable to developers, users, and stakeholders.
  • Documentation of selected explainability techniques (e.g., SHAP, LIME)
  • Initial drafts of user guidance materials and FAQs
  • Architecture diagrams and model cards
 Design for Clarity: Build and document explainability features as part of the core system architecture. Prepare initial documentation to explain system logic.
Accountability Establish clear lines of ownership for the design, development and approval of the AI system.
  • Approved role/responsibility matrix (e.g., RACI) for the project team
  • Documented sign-offs at key development milestones
  • Traceability logs from requirements to code
 Assign Ownership: Finalize and get approval for the RACI chart before the main development phase begins. Ensure all key design and testing decisions are logged and attributed.
Data Governance, Quality and Privacy Engineer the system to incorporate robust data governance frameworks, ensure high data quality throughout its lifecycle, and protect personal data by design and by default. This includes establishing clear policies for data collection, storage, access, and usage, implementing mechanisms for data accuracy and integrity, and prioritizing privacy-preserving techniques from inception.
  • Completed Data Privacy Impact Assessment (DPIA) before deployment
  • Documentation of data minimization and de-identification techniques used
  • System architecture showing privacy-preserving data flows
 Implement Privacy by Design: Finalize the DPIA based on the system's architecture. Implement and test data minimization and de-identification scripts and processes.
Human Oversight Design workflows that require human intervention for high-risk decisions and provide clear interfaces for interaction.
  • System design documents detailing human review workflows
  • Defined criteria for when human intervention is required
  • Usability testing reports on human oversight interfaces
 Design Intervention Points: Define and document the specific triggers and criteria for mandatory human review. Build and test the user interfaces for oversight.
Technical Robustness and Security (EU AI Act Art 15) Build an AI system that meets specified levels of accuracy, robustness, and cybersecurity.
  • Accuracy benchmarks established during testing
  • Results of robustness tests (e.g., adversarial inputs, fault tolerance)
  • Cybersecurity penetration test reports from the development environment.
  • Documented safety-specific risk assessments for the design
  • Records of fail-safe mechanism testing in a development environment
  • Architectural documents detailing safety features
 Validate Technical Performance: Establish and document accuracy baselines. Perform adversarial and fault tolerance testing to ensure resilience. Conduct pre-deployment security testing. Engineer for Safety: Conduct a safety-focused risk assessment during the design phase. Code and unit-test all safety-critical features and fail-safe mechanisms.
Explainability Ensure that the system provides meaningful explanations of outputs in a way that matches the user's technical level. Go beyond transparency - focus on interpretability for non-technical users.
  • Results of explainability validation tests using representative user personas
  • Internal testing of explanation outputs across key use cases
  • UX research records (e.g., user testing sessions on explanation clarity)
 Design for Interpretability: Select, implement, and test explanation techniques suitable for different user types (technical and non-technical). Document how the system generates explanations and validate them with sample users before release.
Environmental Sustainability Design the system to minimize energy consumption during both training and inference phases, and consider the overall carbon footprint of data and compute resources.
  • Energy consumption logs during model training and validation
  • Records of optimization activities (e.g., model pruning, distillation)
  • Carbon footprint estimates from cloud or infrastructure providers
  • Documentation of compute resource allocation decisions
 Optimize for Efficiency: Monitor energy use during model training. Apply efficiency techniques where feasible (e.g., smaller models, batch processing). Document and track sustainability KPIs in the design phase.
Reliability and Availability Engineer the system for consistent performance and uptime, especially for safety-critical or business-critical use cases.
  • Results from load testing, stress testing, and failure mode analysis
  • Documentation of redundancy and failover mechanisms
  • Test logs from system reliability and availability scenarios
 Design for Resilience: Implement and test failover, load balancing, and redundancy mechanisms during development. Simulate failure scenarios and validate recovery times before deployment.
Continuous Monitoring and Feedback Loops Implement processes and tools for ongoing monitoring of model performance, drift, bias emergence, and unintended consequences once deployed.
  • Design documentation showing monitoring hooks and logging frameworks
  • Testing of performance monitoring tools during development
  • Records of alert threshold definitions for production monitoring
 Build Monitoring Hooks: Integrate monitoring and telemetry features into the system architecture. Define performance thresholds, bias alerts, and drift detection triggers as part of the development process.
Ethical and Societal Impact Awareness Proactively assess and mitigate broader societal, ethical, and economic impacts especially in high-risk use cases like recruitment, healthcare, finance, or criminal justice.
  • Completed Ethical Impact Assessment (EIA) for the system design
  • Documentation of mitigation steps for identified ethical risks
  • Stakeholder consultation records during development
 Run Impact Assessments: Conduct and document an ethical risk assessment before finalizing the system design. Review findings with stakeholders and adjust the system to mitigate identified risks.
AI Lifecycle Management and Documentation Maintain detailed, auditable documentation for every stage of the AI lifecycle from data sourcing and model training to deployment and decommissioning (helps with both compliance and knowledge transfer).
  • Version-controlled records of model development stages
  • Data lineage documentation
  • Change management logs
  • Records of peer reviews and approvals at key lifecycle stages
 Document the Lifecycle: Maintain comprehensive records for each development milestone (data sourcing, model training, testing, deployment plans). Ensure documentation is structured and accessible for audits and future knowledge transfer.
