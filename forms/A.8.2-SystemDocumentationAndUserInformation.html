A.8.2 System Documentation and User Information

This document outlines the requirements for system documentation, information for interested parties, external reporting, and incident communication plans related to AI systems.

1. System Documentation for Users (A.8.2)

Field Description
AI System Name Name or ID of the system
User Types Who are the end users? (e.g., internal analysts, customers, auditors)
User Guidance Materials What information is provided? (e.g., user manual, model explainability guide)
Delivery Format How is it delivered? (PDF, embedded help, chatbot, live training)
Access Location Where can users find documentation? (e.g., SharePoint, web app help section)
Last Updated Date of last content update
Owner Who maintains this documentation?

2. Information for Interested Parties (A.8.5)

Party Type Stakeholder Name / Group Required Information Frequency Delivery Channel Notes
Regulator Data Protection Authority Data usage summary, incident logs Quarterly Secure portal
Internal Oversight AI Ethics Committee Use case approval logs, fairness reports Monthly Email + dashboard
Customers Enterprise Clients Model logic summary, risk disclaimers Onboarding + update cycle Web UI, Docs page

3. External Reporting Requirements (A.8.3)

Requirement Type Description Trigger/Schedule Destination Format Owner
Regulatory Reporting Mandatory AI system registration or updates Launch, major update National AI register JSON, XML, Form Compliance Officer
Model Risk Reporting Periodic reporting for high-risk AI models Quarterly Supervisory Board PDF Model Risk Team
Public Transparency Voluntary public summary of system purpose & safeguards Launch + annually Company website HTML/PDF Communications Lead

4. Incident Communication Plan (A.8.4)

Incident Type Description Audience Timeframe Communication Method Responsible Role
Data Breach Unintended exposure of training data DPO, Regulators, Affected users Within 72 hours Email, regulatory portal CISO
Model Misuse AI model used outside intended scope Internal Oversight, Legal ASAP Slack, email, incident ticket AI Ops Manager
Model Failure False predictions causing harm End users, Customer support <24 hours In-app alert, support notice Product Owner
Regulatory Trigger New legal obligations Compliance, Legal As needed Briefing docs, task force Compliance Lead
