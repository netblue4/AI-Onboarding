1. Intended Use Definition (A.9.4)

This document defines the intended use and limitations of an AI system, ensuring alignment with responsible AI principles and regulatory compliance.

Field Description
AI System Name Name or ID of the AI application
Business Purpose What specific problem or task does this system address?
Target Users / Stakeholders Who will use this system or be affected by it? (e.g., employees, customers, analysts)
Intended Use Describe the legitimate and approved use cases (include examples)
Out-of-Scope Use / Limitations What is this system not intended to do? (e.g., no use for employment decisions)
Operational Boundaries Context and constraints (e.g., only operates on internal data, only usable with human review)
